# Reflection

## What I Learned
During this practical, I learned how to refactor code by organizing tasks into functions. I now understand the importance of separating concerns, which helps in making the code more readable and reusable.

## Challenges I Faced
One challenge I faced was ensuring that the functions followed the Single Responsibility Principle (SRP). Initially, I had functions that were doing too much, but after some feedback, I was able to break them down into smaller tasks.

## Improvements
Looking back, I could have structured some of my code more efficiently. For instance, I used repeated code in certain areas where I could have reused functions, and next time I will try to avoid this.

## Practical Application
These refactoring techniques will help me in future projects where code readability and maintainability are essential, especially in larger projects. I'll make sure to always write code that is easy to debug and modify.
